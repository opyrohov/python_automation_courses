Part 2

Virtual Environments

Isolate project dependencies

Why Virtual Environments?

The Problem: Different projects need different package versions

Without virtual environments:

  • Project A needs pytest 7.0
  • Project B needs pytest 8.0
  • Both installed globally = CONFLICT! üí•

With virtual environments:

  • Each project has its own isolated environment
  • No conflicts between projects
  • Clean, reproducible setups
Best Practice: Always use a virtual environment for your projects!

Creating a Virtual Environment

# Navigate to your project folder cd my_automation_project # Create virtual environment (named 'venv') python -m venv venv # On Windows: Activate it venv\Scripts\activate # On Mac/Linux: Activate it source venv/bin/activate # You'll see (venv) in your terminal prompt (venv) C:\my_automation_project>
venv folder: Contains Python interpreter and all installed packages for this project

Working with Virtual Environments

# After activation, install packages pip install playwright pytest # Check installed packages pip list # Deactivate when done deactivate # Next time you work on this project: # 1. Activate the venv venv\Scripts\activate # 2. Start coding!
Remember: Always activate your venv before working on your project!

Don't Commit Virtual Environment!

Important: Never commit the venv folder to Git!

Create a .gitignore file:

# .gitignore venv/ __pycache__/ *.pyc .pytest_cache/ .env *.log screenshots/ test-results/
Why? venv is large and machine-specific. Other developers create their own venv.

Part 5

Introduction to Testing Mindset

Think like a tester

What is Software Testing?

Testing: Verifying that software works as expected and finding bugs before users do

Manual Testing: Human clicks through the app

  • ‚úÖ Good for exploratory testing
  • ‚ùå Slow, repetitive, error-prone

Automated Testing: Code tests the app

  • ‚úÖ Fast, reliable, repeatable
  • ‚úÖ Runs overnight, on every code change
  • ‚ùå Requires initial setup time

Types of Automated Tests

Unit Tests

Test individual functions in isolation

def test_add(): assert add(2, 3) == 5
Integration Tests

Test multiple components together

def test_api_flow(): # Test API + DB
E2E (End-to-End) Tests: Test the entire user flow through the UI
‚Üê This is what we'll do with Playwright!

The Testing Pyramid

E2E Tests (Few)
Integration Tests (Some)
Unit Tests (Many)
Strategy: Lots of fast unit tests, fewer slow E2E tests

What Makes a Good Test?

‚úÖ FAST: Runs quickly so you can run often
‚úÖ INDEPENDENT: Doesn't depend on other tests
‚úÖ REPEATABLE: Same result every time
‚úÖ CLEAR: Easy to understand what it tests
‚úÖ FOCUSED: Tests one thing at a time

AAA Pattern for Tests

def test_successful_login(page): # ARRANGE - Set up test data and preconditions username = "test@example.com" password = "SecurePass123" page.goto("https://example.com/login") # ACT - Perform the action being tested page.fill("#username", username) page.fill("#password", password) page.click("button[type='submit']") # ASSERT - Verify the expected outcome assert page.is_visible(".dashboard") assert "Welcome" in page.text_content(".user-greeting")
AAA = Arrange, Act, Assert - Standard test structure

Setting Up Your First Automation Project

Let's build it together!

Step 1: Create Project Structure

# Create project folder mkdir my_first_automation cd my_first_automation # Create folder structure mkdir tests mkdir pages mkdir utils mkdir test_data # Create empty files # Windows: type nul > tests\__init__.py type nul > pages\__init__.py type nul > utils\__init__.py # Mac/Linux: touch tests/__init__.py touch pages/__init__.py touch utils/__init__.py
__init__.py: Makes folders into Python packages

Step 2: Set Up Virtual Environment

# Create virtual environment python -m venv venv # Activate it (Windows) venv\Scripts\activate # Activate it (Mac/Linux) source venv/bin/activate # Verify activation - you should see (venv) (venv) C:\my_first_automation>
‚úì Virtual environment ready! Now we can install packages.

Step 3: Create requirements.txt

# Create requirements.txt file with these contents: playwright==1.40.0 pytest==8.0.0 pytest-playwright==0.4.4 python-dotenv==1.0.0

What each package does:

  • playwright: Browser automation library
  • pytest: Testing framework
  • pytest-playwright: Pytest integration for Playwright
  • python-dotenv: Load environment variables from .env file

Step 4: Install Dependencies

# Install all packages from requirements.txt pip install -r requirements.txt # This might take a minute... # You'll see packages being downloaded and installed # After installation completes: # Install Playwright browsers (Chrome, Firefox, Safari) playwright install # Verify installation pytest --version playwright --version
‚úì All dependencies installed! Ready to write tests.

Step 5: Create .gitignore

# .gitignore - files/folders to exclude from Git # Virtual environment venv/ env/ # Python cache __pycache__/ *.pyc *.pyo # Pytest cache .pytest_cache/ # Test results test-results/ screenshots/ videos/ # Environment variables .env # IDE settings .vscode/ .idea/

Step 6: Create pytest.ini

# pytest.ini - Pytest configuration [pytest] # Where to look for tests testpaths = tests # Markers for organizing tests markers = smoke: Quick smoke tests regression: Full regression suite login: Login-related tests # Output options addopts = -v --tb=short
Usage: Run specific test groups with pytest -m smoke

Step 7: Create Configuration

# utils/config.py import os from dotenv import load_dotenv # Load environment variables from .env file load_dotenv() # Base URL for tests BASE_URL = os.getenv("BASE_URL", "https://example.com") # Browser settings HEADLESS = os.getenv("HEADLESS", "False") == "True" BROWSER = os.getenv("BROWSER", "chromium") # Timeouts (in milliseconds) DEFAULT_TIMEOUT = 30000 NAVIGATION_TIMEOUT = 60000

Step 8: Create .env File

# .env - Environment-specific variables (don't commit!) BASE_URL=https://example.com HEADLESS=False BROWSER=chromium # Test credentials (never commit real credentials!) TEST_USERNAME=test@example.com TEST_PASSWORD=TestPassword123
Security: Never commit .env to Git! It's in .gitignore.

Your First Test - test_example.py

# tests/test_example.py import pytest from playwright.sync_api import Page def test_google_search(page: Page): # Navigate to Google page.goto("https://www.google.com") # Verify page loaded assert page.title() == "Google" # Search for something page.fill("textarea[name='q']", "Playwright Python") page.press("textarea[name='q']", "Enter") # Wait for results page.wait_for_selector("#search") # Verify results appeared assert page.is_visible("#search")

Preparing for Playwright

Installation checklist

Playwright Installation Checklist

‚úì Step 1: Python 3.1x installed
‚úì Step 2: Virtual environment created and activated
‚úì Step 3: Playwright installed via pip
‚úì Step 4: Browser drivers installed (playwright install)
‚úì Step 5: pytest and pytest-playwright installed

Verify Your Setup

# Check Python version (should be 3.1x) python --version # Check virtual environment is activated (should see 'venv') # Windows: where python # Should show path inside venv folder # Check pytest is installed pytest --version # Check playwright is installed playwright --version # List installed packages pip list | findstr playwright pip list | findstr pytest

Playwright Browsers

Playwright installs and manages browser versions for you:

Chromium: Open-source version of Chrome (default)
Firefox: Mozilla Firefox
WebKit: Safari's engine (works on Windows/Linux too!)
# Install all browsers playwright install # Install specific browser playwright install chromium playwright install firefox playwright install webkit

Common Setup Issues & Solutions

Issue: "playwright: command not found"
Solution: Make sure venv is activated
Issue: "ModuleNotFoundError: No module named 'playwright'"
Solution: Run pip install playwright
Issue: Tests can't find browsers
Solution: Run playwright install
Issue: Wrong Python version
Solution: Create venv with correct Python: python3.11 -m venv venv

Useful Pytest Commands

# Run all tests pytest # Run specific test file pytest tests/test_login.py # Run specific test function pytest tests/test_login.py::test_successful_login # Run with verbose output pytest -v # Run tests with specific marker pytest -m smoke # Run in headed mode (see browser) pytest --headed # Run with specific browser pytest --browser firefox # Stop on first failure pytest -x
