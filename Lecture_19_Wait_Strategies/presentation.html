Common Issues Without Proper Waits:

Part 1

Auto-Waiting

Playwright's built-in intelligence

Playwright's Auto-Waiting

‚úÖ Playwright automatically waits for elements before actions!

What Playwright Waits For:

  • ‚úì Element is attached to DOM
  • ‚úì Element is visible
  • ‚úì Element is stable (not animating)
  • ‚úì Element receives events (not obscured)
  • ‚úì Element is enabled (for inputs)

Auto-Waiting in Action

# No explicit wait needed! page.locator("#submit-button").click() # Playwright automatically waits for: # 1. Element to be in DOM # 2. Element to be visible # 3. Element to be stable # 4. Element to be enabled # All in one line! # This also auto-waits page.locator("#username").fill("user@example.com") # Even assertions auto-wait expect(page.locator("#message")).to_be_visible()

When Auto-Waiting Happens

# Actions that auto-wait: page.locator("#btn").click() # Waits for actionability page.locator("#input").fill("text") # Waits for enabled + visible page.locator("#checkbox").check() # Waits for actionable page.locator("#link").hover() # Waits for visible + stable # Assertions that auto-wait: expect(page.locator("#el")).to_be_visible() expect(page.locator("#el")).to_have_text("Hello") expect(page.locator("#el")).to_be_enabled() # Getting properties does NOT auto-wait: text = page.locator("#el").text_content() # Immediate count = page.locator(".item").count() # Immediate

Part 2

wait_for_selector()

Explicit element waiting

wait_for_selector() - Basic Usage

# Wait for element to appear page.wait_for_selector("#dynamic-content") # Wait with timeout (default is 30 seconds) page.wait_for_selector("#slow-element", timeout=10000) # 10 seconds # Wait for element to be visible (default) page.wait_for_selector("#element", state="visible") # Wait for element to be hidden page.wait_for_selector("#loading", state="hidden") # Wait for element to be attached (in DOM) page.wait_for_selector("#element", state="attached")

wait_for_selector() States

attached

Element is in DOM (may not be visible)

page.wait_for_selector( "#element", state="attached" )

visible (default)

Element is visible on page

page.wait_for_selector( "#element", state="visible" )

hidden

Element is not visible or not in DOM

page.wait_for_selector( "#loading", state="hidden" )

detached

Element is removed from DOM

page.wait_for_selector( "#element", state="detached" )

Practical wait_for_selector() Examples

# Wait for loading spinner to disappear page.wait_for_selector(".loading-spinner", state="hidden") # Wait for success message to appear page.wait_for_selector(".success-message", state="visible") # Wait for dynamic content after AJAX call page.locator("#load-btn").click() page.wait_for_selector(".dynamic-content", timeout=5000) # Wait for modal to close page.locator(".close-modal").click() page.wait_for_selector(".modal", state="detached")

Part 3

Load States

Waiting for page readiness

wait_for_load_state()

# Wait for page to load (default: "load") page.goto("https://example.com") page.wait_for_load_state() # Wait for DOM to be ready page.wait_for_load_state("domcontentloaded") # Wait for network to be idle page.wait_for_load_state("networkidle") # Combined example page.goto("https://example.com") page.wait_for_load_state("networkidle") # Wait for all network requests print("Page fully loaded with all resources")

Load State Types

load (default)

Waits for the 'load' event - page and resources loaded

page.wait_for_load_state("load")

domcontentloaded

Waits for DOM to be ready - faster, doesn't wait for images/stylesheets

page.wait_for_load_state("domcontentloaded")

networkidle

Waits for no network connections for 500ms - good for SPAs

page.wait_for_load_state("networkidle")

When to Use Each Load State

Use domcontentloaded

  • Simple pages
  • When you don't need images
  • Faster tests
page.goto(url) page.wait_for_load_state( "domcontentloaded" )

Use networkidle

  • Single Page Apps (SPAs)
  • AJAX-heavy pages
  • Dynamic content loading
page.goto(url) page.wait_for_load_state( "networkidle" )

Part 4

Timeouts

When and when NOT to use them

wait_for_timeout() - The Anti-Pattern

‚ö†Ô∏è Avoid wait_for_timeout() When Possible!

Using fixed sleeps makes tests slow and unreliable

‚ùå BAD

# Hard-coded wait page.locator("#btn").click() page.wait_for_timeout(3000) # Hope element is ready...

‚úÖ GOOD

# Wait for condition page.locator("#btn").click() page.wait_for_selector( "#result", state="visible" )

When wait_for_timeout() Is Acceptable

Rare Valid Use Cases:

  • üé¨ Animation needs to complete (no other way to detect)
  • üêõ Debugging (temporary only!)
  • ‚è±Ô∏è Testing time-based behavior
# Example: CSS animation must complete page.locator("#animate-btn").click() page.wait_for_timeout(500) # Known animation duration # Debugging only - REMOVE BEFORE COMMIT page.wait_for_timeout(5000) # TODO: Replace with proper wait # Testing time-based feature start_time = time.time() page.wait_for_timeout(2000) elapsed = time.time() - start_time assert elapsed >= 2

Configuring Timeouts

# Global timeout for all actions (default: 30 seconds) page.set_default_timeout(60000) # 60 seconds # Per-action timeout page.locator("#slow-btn").click(timeout=45000) # Navigation timeout page.goto("https://example.com", timeout=60000) # Assertion timeout expect(page.locator("#msg")).to_be_visible(timeout=10000) # Wait for selector with custom timeout page.wait_for_selector("#element", timeout=5000)

Part 5

Custom Wait Conditions

Wait for anything

wait_for_function() - Custom Conditions

# Wait for custom JavaScript condition page.wait_for_function("() => document.readyState === 'complete'") # Wait for element count page.wait_for_function( "() => document.querySelectorAll('.item').length > 10" ) # Wait for text content page.wait_for_function( "selector => document.querySelector(selector).textContent.includes('Success')", arg="#message" ) # Wait for style change page.wait_for_function( "el => getComputedStyle(el).opacity === '1'", arg=page.locator("#element").element_handle() )

Advanced Custom Waits

# Wait for URL to change page.wait_for_function( "url => window.location.href.includes(url)", arg="dashboard" ) # Wait for local storage page.wait_for_function( "key => localStorage.getItem(key) !== null", arg="user_token" ) # Wait for variable to be defined page.wait_for_function("() => typeof myApp !== 'undefined'") # Wait for AJAX to complete (jQuery) page.wait_for_function("() => jQuery.active === 0")

Combining Wait Strategies

# Complete example: Form submission def submit_form_and_wait(page): # 1. Fill form page.locator("#email").fill("user@example.com") page.locator("#password").fill("password123") # 2. Submit page.locator("#submit").click() # 3. Wait for loading to disappear page.wait_for_selector(".loading", state="hidden") # 4. Wait for success message page.wait_for_selector(".success-message", state="visible") # 5. Wait for network to settle page.wait_for_load_state("networkidle") # 6. Verify URL changed expect(page).to_have_url("*/dashboard")

Wait Strategy Best Practices

‚úÖ DO

  • Trust auto-waiting for actions
  • Use expect() for assertions
  • Wait for specific conditions
  • Use wait_for_selector() states
  • Choose appropriate load states
  • Set reasonable timeouts

‚ùå DON'T

  • Use wait_for_timeout() by default
  • Add unnecessary waits
  • Use tiny timeouts (< 1s)
  • Ignore timeout errors
  • Mix sync/async incorrectly
  • Wait when auto-wait works

Decision Tree: Which Wait to Use?

Need to interact with element?

‚Üí Use auto-waiting (just click/fill/check)

Need to verify element state?

‚Üí Use expect() assertions

Waiting for element to appear/disappear?

‚Üí Use wait_for_selector() with state

Waiting for page to load?

‚Üí Use wait_for_load_state()

Need custom condition?

‚Üí Use wait_for_function()

Common Wait Patterns

# Pattern 1: Wait for loading to finish page.wait_for_selector(".spinner", state="hidden") # Pattern 2: Wait for element before interaction page.wait_for_selector("#button", state="visible") page.locator("#button").click() # Pattern 3: Wait after navigation page.goto(url) page.wait_for_load_state("networkidle") # Pattern 4: Wait for dynamic content page.locator("#search").fill("playwright") page.wait_for_selector(".search-results", state="visible") # Pattern 5: Wait for element to disappear page.locator("#delete").click() page.wait_for_selector(".item", state="detached")

Troubleshooting Wait Issues

TimeoutError: Timeout 30000ms exceeded

Solutions:

  • Check selector is correct
  • Verify element actually appears
  • Increase timeout if legitimately slow
  • Use correct wait state

Element not visible/clickable

Solutions:

  • Wait for element to be visible first
  • Scroll element into view
  • Wait for animations to complete
  • Check for overlapping elements
