What is Inheritance?

Inheritance allows a class to inherit attributes and methods from another class.

Terms:

  • Parent Class: Base/Super class
  • Child Class: Derived/Sub class
  • Inherits: Gets parent's code

Benefits:

  • ‚úÖ Code reuse
  • ‚úÖ Organization
  • ‚úÖ Maintainability
  • ‚úÖ Extensibility
üí° Key: Child classes inherit everything from parent!

Basic Inheritance Syntax

# Parent class class Animal: def __init__(self, name): self.name = name def speak(self): print(f"{self.name} makes a sound") # Child class inherits from Animal class Dog(Animal): # ‚Üê Inheritance! def speak(self): print(f"{self.name} says Woof!") dog = Dog("Buddy") dog.speak() # Buddy says Woof!
Syntax: class ChildClass(ParentClass):

The super() Function

super() calls methods from the parent class.

class Person: def __init__(self, name, age): self.name = name self.age = age class Employee(Person): def __init__(self, name, age, employee_id): super().__init__(name, age) # Call parent! self.employee_id = employee_id emp = Employee("Alice", 30, "EMP001") print(emp.name) # Alice

Why Use super()?

‚ùå Without super():

def __init__(self, name, age, emp_id):
    self.name = name     # Duplicate
    self.age = age       # Duplicate
    self.employee_id = emp_id

‚úÖ With super():

def __init__(self, name, age, emp_id):
    super().__init__(name, age)
    self.employee_id = emp_id
Best Practice: Always use super() to call parent's __init__()

Method Overriding

Child classes can override parent methods.

class Shape: def area(self): return 0 # Default implementation class Rectangle(Shape): def __init__(self, width, height): self.width = width self.height = height def area(self): # Override parent's area() return self.width * self.height rect = Rectangle(5, 10) print(rect.area()) # 50

Override vs Extend

Complete Override:

def speak(self): # Replace completely return "Woof!"

Use when behavior is completely different

Extend (use super()):

def speak(self): # Call parent first super().speak() print("Woof!")

Use when adding to parent's behavior

Multiple Child Classes

One parent can have many children!

class Animal: def speak(self): pass class Dog(Animal): def speak(self): return "Woof!" class Cat(Animal): def speak(self): return "Meow!" class Bird(Animal): def speak(self): return "Tweet!"

Multi-Level Inheritance

Inheritance can have multiple levels:

class Animal: # Level 1 pass class Mammal(Animal): # Level 2 pass class Dog(Mammal): # Level 3 pass # Dog inherits from Mammal AND Animal!
‚ö†Ô∏è Warning: Keep inheritance 2-3 levels max. Too deep = hard to maintain!

Why Use Inheritance in Automation?

  • BasePage Pattern: Common page functionality
  • BaseTest Pattern: Common test setup
  • Code Reuse: Write once, use everywhere
  • Consistency: Same behavior across all pages
  • Maintainability: Update in one place
  • Industry Standard: Professional approach

The BasePage Pattern

Create a base class with common page functionality:

class BasePage: """Base class for all page objects.""" def __init__(self, page): self.page = page def click(self, selector): self.page.click(selector) def fill(self, selector, text): self.page.fill(selector, text) def get_text(self, selector): return self.page.locator(selector).text_content()

Creating Page Objects from BasePage

class LoginPage(BasePage): """Login page inherits from BasePage.""" def __init__(self, page): super().__init__(page) self.username_input = "#username" self.password_input = "#password" def login(self, username, password): self.fill(self.username_input, username) # From BasePage! self.fill(self.password_input, password) self.click("#submit") # From BasePage!
‚úÖ LoginPage has all BasePage methods automatically!

BasePage Pattern Benefits

‚úÖ Code Reuse

Write click(), fill(), wait() once ‚Üí use in all pages

‚úÖ Consistency

All pages behave the same way

‚úÖ Maintainability

Update BasePage ‚Üí all pages updated automatically

‚úÖ Error Handling

Add screenshots/logging in one place

Complete BasePage Example

class BasePage: def __init__(self, page): self.page = page def click(self, selector): self.page.click(selector) def fill(self, selector, text): self.page.fill(selector, text) def is_visible(self, selector): return self.page.is_visible(selector) def wait_for_element(self, selector): self.page.wait_for_selector(selector) def screenshot(self, name): self.page.screenshot(path=f"{name}.png")

Using BasePage in Multiple Pages

class HomePage(BasePage): def __init__(self, page): super().__init__(page) self.search_box = "#search" def search(self, query): self.fill(self.search_box, query) # From BasePage class ProductPage(BasePage): def __init__(self, page): super().__init__(page) self.add_to_cart_btn = "#add-to-cart" def add_to_cart(self): self.click(self.add_to_cart_btn) # From BasePage

Best Practice: DRY (Don't Repeat Yourself)

‚ùå Code Duplication:

class HomePage:
    def click_logout(self):
        print("Waiting...")
        self.page.click("#logout")

class ProfilePage:
    def click_logout(self):
        print("Waiting...")  # DUPLICATE!
        self.page.click("#logout")  # DUPLICATE!

‚úÖ Use Inheritance:

class BasePage:
    def click_element(self, selector):
        print("Waiting...")
        self.page.click(selector)

# Both inherit from BasePage - no duplication!

Page Object Specific Best Practices

  • Store selectors in __init__: All selectors in one place
  • Use UPPERCASE for selectors: Easy to identify
  • Return data, don't assert: Let tests do assertions
  • Use private methods: _method_name for internal use
  • One page = one class: Don't combine pages
  • Inherit from BasePage: Always use base class
üí° Remember: Page objects represent user actions, not implementation!
