What is Playwright?

Playwright is a modern browser automation framework created by Microsoft

Key Features:

  • üåê Cross-browser: Chromium, Firefox, WebKit (Safari)
  • ‚ö° Fast & Reliable: Auto-waiting, no flaky tests
  • üîß Developer-friendly: Great API, excellent documentation
  • üì± Mobile support: Test mobile web views
  • üé• Screenshots & Videos: Built-in recording
  • üåç Multiple languages: Python, JavaScript, Java, .NET
Released: 2020 by Microsoft
Status: Active development, production-ready

Part 2

Playwright vs Selenium

Understanding the differences

What is Selenium?

Selenium has been the standard for browser automation since 2004

Characteristics:

  • ‚è∞ Mature: 20+ years old, battle-tested
  • üåç Popular: Huge community, lots of resources
  • üîß WebDriver Protocol: Industry standard (W3C)
  • üìö Many bindings: Python, Java, C#, Ruby, JavaScript
  • üåê Cross-browser: Chrome, Firefox, Safari, Edge
Challenges:
  • Slower execution
  • Flaky tests (timing issues)
  • Complex setup (drivers, versions)
  • Limited built-in features

Playwright vs Selenium: Key Differences

Feature Playwright Selenium
Age 2020 (Modern) 2004 (Mature)
Speed ‚ö° Faster üê¢ Slower
Auto-waiting ‚úÖ Built-in ‚ùå Manual
Browsers Chromium, Firefox, WebKit Chrome, Firefox, Safari, Edge
Setup ‚úÖ Simple (auto-install) ‚ö†Ô∏è Complex (driver mgmt)
Network Control ‚úÖ Full control ‚ùå Limited
Screenshots/Video ‚úÖ Built-in ‚ö†Ô∏è Third-party

When to Use Playwright vs Selenium?

Choose Playwright if:
  • ‚úÖ Starting a new project
  • ‚úÖ Need speed and reliability
  • ‚úÖ Testing modern web applications
  • ‚úÖ Want built-in features (video, screenshots, network)
  • ‚úÖ Need mobile browser emulation
Choose Selenium if:
  • ‚ö†Ô∏è Large existing Selenium test suite
  • ‚ö†Ô∏è Team already expert in Selenium
  • ‚ö†Ô∏è Organizational requirement or policy
  • ‚ö†Ô∏è Need specific Selenium ecosystem tools
üí° Tip: For new projects, Playwright is the modern choice!

Setting up your environment

Prerequisites Check

Before installing Playwright, ensure you have:

‚úÖ Python 3.1x installed

‚úÖ Virtual Environment created and activated

Installation: Step-by-Step

# Step 1: Ensure venv is activated (venv) $ # Step 2: Install Playwright Python package pip install playwright # Step 3: Install pytest and pytest-playwright pip install pytest pytest-playwright # Step 4: Install browser binaries playwright install # Step 5: Verify installation playwright --version pytest --version

What Gets Installed?

Python Packages:

Browser Binaries:

  • üåê Chromium ~300 MB (Chrome/Edge engine)
  • ü¶ä Firefox ~150 MB (Mozilla Firefox)
  • üß≠ WebKit ~50 MB (Safari engine)

Playwright Advantage: Automatically downloads and manages all browser drivers!

Playwright's Three Browsers

1. Chromium (Default)
  • Open-source version of Chrome/Edge
  • Most popular browser engine (~65% market share)
  • Use for: Testing Chrome, Edge, Brave, Opera
2. Firefox
  • Mozilla Firefox browser
  • Different rendering engine (Gecko)
  • Use for: Firefox-specific testing
3. WebKit
  • Safari's rendering engine
  • Works on Windows/Linux too!
  • Use for: Safari/iOS web testing

Installing Browsers

Creating Project Structure

# Create project folder mkdir my_playwright_project cd my_playwright_project # Create folder structure mkdir tests pages utils test_data # Create __init__.py files (Windows) type nul > tests\__init__.py type nul > pages\__init__.py type nul > utils\__init__.py # Create __init__.py files (Mac/Linux) touch tests/__init__.py pages/__init__.py utils/__init__.py # Create virtual environment python -m venv venv venv\Scripts\activate # Windows source venv/bin/activate # Mac/Linux

Create requirements.txt

Part 6

Your First Playwright Script

Opening a browser and navigating

Method 1: Sync API (Simple)

# first_script.py - Simple Playwright script from playwright.sync_api import sync_playwright with sync_playwright() as p: # Launch browser browser = p.chromium.launch(headless=False) # Create new page page = browser.new_page() # Navigate to URL page.goto("https://playwright.dev") # Print page title print(f"Page title: {page.title()}") # Close browser browser.close()
Run it: python first_script.py

Breaking Down the Script

# 1. Import Playwright from playwright.sync_api import sync_playwright
# 2. Create Playwright instance with sync_playwright() as p: # 'p' is the Playwright object # 'with' ensures proper cleanup
# 3. Launch browser browser = p.chromium.launch(headless=False) # headless=False: See the browser window # headless=True: Run in background (default)
# 4. Create page (tab) page = browser.new_page()
# 5. Navigate page.goto("https://playwright.dev")

Method 2: Pytest Test (Recommended)

# tests/test_first.py import pytest from playwright.sync_api import Page, expect def test_playwright_homepage(page: Page): """Test that Playwright homepage loads correctly.""" # ARRANGE url = "https://playwright.dev" # ACT page.goto(url) # ASSERT expect(page).to_have_title("Playwright") expect(page).to_have_url(url) print(f"‚úÖ Successfully navigated to {url}")
Run it: pytest tests/test_first.py

The Magic `page` Fixture

pytest-playwright provides automatic fixtures:
def test_example(page: Page): # 'page' is provided automatically! # No need to launch browser or create page manually page.goto("https://example.com")

What pytest-playwright does for you:

  • ‚úÖ Launches browser before test
  • ‚úÖ Creates fresh page for each test
  • ‚úÖ Takes screenshot on failure
  • ‚úÖ Closes browser after test
  • ‚úÖ Handles cleanup automatically
Result: Cleaner test code, less boilerplate!

Running Your Tests

# Basic run (headless mode) pytest # Run in headed mode (see browser) pytest --headed # Run with specific browser pytest --browser chromium pytest --browser firefox pytest --browser webkit # Run all browsers pytest --browser chromium --browser firefox --browser webkit # Slow down execution (helpful for watching) pytest --headed --slowmo 1000 # Run specific test file pytest tests/test_first.py # Run specific test function pytest tests/test_first.py::test_playwright_homepage

Essential Playwright Commands

# Navigation page.goto("https://example.com") page.go_back() page.go_forward() page.reload() # Get information title = page.title() url = page.url content = page.content() # Screenshots page.screenshot(path="screenshot.png") # Wait for page load page.wait_for_load_state("networkidle") page.wait_for_load_state("domcontentloaded") # Close page.close()

Browser Launch Options

# Headed mode (see browser window) browser = p.chromium.launch(headless=False) # Headless mode (no UI, faster) browser = p.chromium.launch(headless=True) # Slow down operations (for debugging) browser = p.chromium.launch( headless=False, slow_mo=1000 # 1 second delay between actions ) # Custom window size browser = p.chromium.launch( headless=False, args=['--window-size=1920,1080'] ) # Enable devtools browser = p.chromium.launch( headless=False, devtools=True )

Page Creation Options

# Basic page page = browser.new_page() # Page with custom viewport page = browser.new_page( viewport={'width': 1920, 'height': 1080} ) # Mobile emulation page = browser.new_page( viewport={'width': 375, 'height': 667}, user_agent='Mozilla/5.0 (iPhone; CPU iPhone OS 14_0 like Mac OS X)' ) # Enable video recording page = browser.new_page( record_video_dir="videos/" ) # Set default timeout page.set_default_timeout(30000) # 30 seconds

Complete Working Example

# tests/test_google_search.py from playwright.sync_api import Page, expect def test_google_search(page: Page): """Test Google search functionality.""" # Navigate to Google page.goto("https://www.google.com") # Verify page loaded expect(page).to_have_title("Google") # Verify search box is visible search_box = page.locator("textarea[name='q']") expect(search_box).to_be_visible() # Type search query search_box.fill("Playwright Python") # Press Enter search_box.press("Enter") # Wait for results page.wait_for_load_state("networkidle") # Verify results page loaded expect(page).to_have_url("**/search?**") print("‚úÖ Google search test passed!")

Troubleshooting Common Issues

Issue: "playwright: command not found"
Solution: Activate virtual environment
venv\Scripts\activate # Windows source venv/bin/activate # Mac/Linux
Issue: "No module named 'playwright'"
Solution: Install playwright
pip install playwright pytest-playwright
Issue: "Executable doesn't exist"
Solution: Install browser binaries
playwright install
Issue: Tests fail with timeout
Solution: Increase timeout or check network
page.set_default_timeout(60000) # 60 seconds
