Your First Function

# Define a function def greet(): print("Hello, World!") # Call the function greet() # Output: Hello, World!
Syntax:
  • def keyword starts the definition
  • Function name (lowercase with underscores)
  • Parentheses () - even if no parameters
  • Colon : to end the definition line
  • Indented body (4 spaces)

Functions with Parameters

# Function with ONE parameter def greet_user(name): print(f"Hello, {name}!") greet_user("Alice") # Hello, Alice! greet_user("Bob") # Hello, Bob! # Function with MULTIPLE parameters def introduce(name, age): print(f"My name is {name} and I'm {age} years old") introduce("Alice", 25)
Parameters vs Arguments:
Parameter = Variable in function definition
Argument = Actual value passed to function

Return Values

# Function that returns a value def add(a, b): return a + b # Use the return value result = add(5, 3) print(result) # 8 # Use directly in expressions total = add(10, 20) + add(5, 15) print(total) # 50
üí° Key Point: return sends a value back to the caller and exits the function

Multiple Return Values

# Return multiple values as a tuple def get_user_info(): name = "Alice" age = 25 city = "New York" return name, age, city # Unpack the return values user_name, user_age, user_city = get_user_info() print(user_name) # Alice print(user_age) # 25 print(user_city) # New York # Or get as tuple user_data = get_user_info() print(user_data) # ('Alice', 25, 'New York')

Default Parameters

# Function with default parameter def greet(name, greeting="Hello"): print(f"{greeting}, {name}!") # Use default greet("Alice") # Hello, Alice! # Override default greet("Bob", "Hi") # Hi, Bob! greet("Charlie", "Good morning") # Good morning, Charlie!
üí° Best Practice: Put required parameters first, then optional parameters with defaults

Keyword Arguments

def create_user(name, email, age, is_admin=False): print(f"Name: {name}") print(f"Email: {email}") print(f"Age: {age}") print(f"Admin: {is_admin}") # Positional arguments (order matters) create_user("Alice", "alice@test.com", 25) # Keyword arguments (order doesn't matter!) create_user(email="bob@test.com", name="Bob", age=30) # Mix (positional first, then keyword) create_user("Charlie", "charlie@test.com", age=35, is_admin=True)

Variable Arguments: *args

# Accept any number of positional arguments def sum_all(*numbers): total = 0 for num in numbers: total += num return total print(sum_all(1, 2, 3)) # 6 print(sum_all(10, 20, 30, 40)) # 100 print(sum_all(5)) # 5
*args: Collects extra positional arguments into a tuple

Variable Keyword Arguments: **kwargs

# Accept any number of keyword arguments def print_config(**settings): for key, value in settings.items(): print(f"{key}: {value}") print_config(browser="Chrome", timeout=30, headless=True) # Output: # browser: Chrome # timeout: 30 # headless: True
**kwargs: Collects extra keyword arguments into a dictionary

Early Return Pattern

def divide(a, b): # Check for error condition first if b == 0: print("Error: Cannot divide by zero!") return None # Normal case return a / b result1 = divide(10, 2) # 5.0 result2 = divide(10, 0) # None (with error message)
üí° Best Practice: Handle edge cases and errors first, then the main logic

Function Scope

# Global variable counter = 0 def increment_local(): # Local variable (only exists in function) local_counter = 0 local_counter += 1 print(f"Local: {local_counter}") increment_local() # 1 increment_local() # 1 (resets each time!) def increment_global(): global counter # Access global variable counter += 1 print(f"Global: {counter}") increment_global() # 1 increment_global() # 2

Documenting Functions with Docstrings

def calculate_area(width, height): """ Calculate the area of a rectangle. Args: width: The width of the rectangle height: The height of the rectangle Returns: The area (width * height) """ return width * height # Access docstring print(calculate_area.__doc__)
üí° Best Practice: Document complex functions with docstrings

Lambda Functions (Quick Functions)

# Regular function def add(a, b): return a + b # Lambda function (one-liner) add_lambda = lambda a, b: a + b print(add(5, 3)) # 8 print(add_lambda(5, 3)) # 8 # Common use: sorting users = [ {"name": "Alice", "age": 25}, {"name": "Bob", "age": 20}, {"name": "Charlie", "age": 30} ] sorted_users = sorted(users, key=lambda u: u["age"])

def navigate_to(url, timeout=30): """Navigate to URL with timeout.""" print(f"Navigating to: {url}") page.goto(url, timeout=timeout * 1000) page.wait_for_load_state("networkidle") return True def click_element(selector, wait=True): """Click element with optional wait.""" if wait: page.wait_for_selector(selector) page.click(selector) return True def fill_field(selector, value): """Fill a form field.""" page.fill(selector, value) return True

Automation Example: Verification Helpers

def verify_text(text): """Verify text is present on page.""" try: page.wait_for_selector(f"text={text}") return True except: return False def verify_element_visible(selector): """Verify element is visible.""" return page.is_visible(selector) def verify_url_contains(expected): """Verify URL contains expected string.""" current_url = page.url return expected in current_url # Usage assert verify_text("Welcome") assert verify_element_visible("#nav-menu") assert verify_url_contains("/dashboard")

Automation Example: Wait Helpers

def wait_for_element(selector, timeout=10): """Wait for element to appear.""" page.wait_for_selector(selector, timeout=timeout * 1000) return True def wait_for_text(text, timeout=10): """Wait for text to appear.""" page.wait_for_selector(f"text={text}", timeout=timeout * 1000) return True def wait_and_click(selector, timeout=10): """Wait for element then click it.""" wait_for_element(selector, timeout) page.click(selector) return True # Usage wait_and_click("#submit-button")

Data-Driven Tests

def run_test_with_data(test_data_list, test_function): """Run tests with multiple data sets.""" passed = 0 failed = 0 for i, data in enumerate(test_data_list, 1): print(f"Test Case {i}") try: test_function(data) passed += 1 except Exception as e: print(f"Failed: {e}") failed += 1 return {"passed": passed, "failed": failed} # Test data test_data = [ {"username": "user1@test.com", "password": "Pass1"}, {"username": "user2@test.com", "password": "Pass2"} ]

Function Best Practices

  • Single Responsibility: Each function should do ONE thing well
  • Descriptive Names: Use clear, verb-based names (click_button, verify_text)
  • Keep It Short: Functions should be 10-20 lines ideally
  • Use Docstrings: Document complex functions
  • Return Values: Return meaningful values (True/False, data, None)
  • Handle Errors: Use early returns for error cases
  • Consistent Style: Follow project naming conventions

Function Naming Conventions

‚úÖ Good Names

calculate_total() send_email() validate_user_input() get_user_by_id() is_valid_email() has_permission()

‚ùå Bad Names

func1() doStuff() x() calculate() process() data()
üí° Pattern: verb_noun() or action_target()

When to Create a Function?

Create a function when:
  • ‚úÖ Code is repeated 2+ times
  • ‚úÖ Code does one clear task
  • ‚úÖ You want to test something in isolation
  • ‚úÖ Code is complex and needs a name
  • ‚úÖ You want to reuse logic across tests
Don't over-engineer:
  • ‚ùå Don't create functions for single-use code
  • ‚ùå Don't create functions that are 1-2 lines
  • ‚ùå Don't create functions that are too generic
