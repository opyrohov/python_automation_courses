Part 1

Why Pytest?

From scripts to real tests

Problems with Scripts

Manual scripts:

  • No automatic PASS / FAIL reporting
  • Must open/close browser manually
  • No test isolation
  • Can't run specific tests
  • No parallel execution
  • No retry on failure

Pytest Gives You

With pytest:

  • ‚úÖ Automatic PASS / FAIL
  • ‚úÖ Browser managed automatically (fixtures)
  • ‚úÖ Each test is isolated
  • ‚úÖ Run tests by name, marker, file
  • ‚úÖ Parallel execution
  • ‚úÖ Retry, reports, plugins

Installation

# Install pytest and Playwright plugin pip install pytest pytest-playwright # Install browsers (if not done) playwright install

‚úÖ pytest-playwright gives you page, context, browser fixtures automatically!

Part 2

Basic Test Functions

Writing your first pytest tests

Your First Pytest Test

# test_example.py from playwright.sync_api import Page def test_page_title(page: Page): """Test that page has correct title.""" page.goto("https://the-internet.herokuapp.com") assert page.title() == "The Internet"

Key rules:

  • File starts with test_
  • Function starts with test_
  • page parameter = automatic fixture

Before vs After

‚ùå Before (manual script):

with sync_playwright() as p: browser = p.chromium.launch() page = browser.new_page() page.goto("https://example.com") print(page.title()) # Manual check browser.close()

‚úÖ After (pytest):

def test_title(page): page.goto("https://example.com") assert page.title() == "Example Domain"

Multiple Tests in One File

# test_login.py from playwright.sync_api import Page def test_login_page_loads(page: Page): page.goto("https://the-internet.herokuapp.com/login") assert page.locator("h2").text_content() == "Login Page" def test_successful_login(page: Page): page.goto("https://the-internet.herokuapp.com/login") page.locator("#username").fill("tomsmith") page.locator("#password").fill("SuperSecretPassword!") page.locator("button[type='submit']").click() assert "/secure" in page.url def test_failed_login(page: Page): page.goto("https://the-internet.herokuapp.com/login") page.locator("#username").fill("wrong") page.locator("#password").fill("wrong") page.locator("button[type='submit']").click() assert "Your username is invalid!" in page.locator("#flash").text_content()

Test Discovery Rules

Pytest automatically finds tests that match these rules:

Rule Example
File starts with test_ test_login.py ‚úÖ
Function starts with test_ def test_title(): ‚úÖ
Class starts with Test class TestLogin: ‚úÖ
Method starts with test_ def test_valid(): ‚úÖ

‚ö†Ô∏è login_test.py or def check_login() will NOT be found!

Part 3

Playwright Fixtures

Automatic browser management

Built-in Playwright Fixtures

Fixture What It Provides Scope
page New page (tab) for each test per test
context Browser context (like incognito) per test
browser Browser instance per session

‚úÖ Each test gets a fresh page - complete isolation!

The page Fixture

def test_navigation(page): """page = fresh browser tab, auto-created.""" page.goto("https://example.com") assert page.title() == "Example Domain" # page is automatically closed after test! def test_another(page): """This gets a DIFFERENT page - fresh state!""" page.goto("https://example.com") # No cookies/data from previous test

No need to create browser, create page, or close browser. Pytest does it all!

Custom Fixtures (conftest.py)

# conftest.py import pytest from playwright.sync_api import Page @pytest.fixture def login_page(page: Page): """Navigate to login page before test.""" page.goto("https://the-internet.herokuapp.com/login") return page @pytest.fixture def logged_in_page(page: Page): """Login and return authenticated page.""" page.goto("https://the-internet.herokuapp.com/login") page.locator("#username").fill("tomsmith") page.locator("#password").fill("SuperSecretPassword!") page.locator("button[type='submit']").click() return page

Using Custom Fixtures

# test_secure.py def test_login_page_has_form(login_page): """login_page fixture navigates for us.""" assert login_page.locator("#username").is_visible() assert login_page.locator("#password").is_visible() def test_secure_area(logged_in_page): """logged_in_page fixture logs in for us.""" assert "/secure" in logged_in_page.url heading = logged_in_page.locator("h2") assert heading.text_content() == " Secure Area"

‚úÖ Fixtures eliminate duplicate setup code!

Part 4

Parameterized Tests

Test multiple inputs with one function

Problem: Duplicate Tests

‚ùå Without parametrize:

def test_checkbox_1(page): page.goto(URL) page.locator("#checkbox1").check() assert page.locator("#checkbox1").is_checked() def test_checkbox_2(page): # Same logic! page.goto(URL) page.locator("#checkbox2").check() assert page.locator("#checkbox2").is_checked()

Solution: @pytest.mark.parametrize

import pytest @pytest.mark.parametrize("username,password,expected", [ ("tomsmith", "SuperSecretPassword!", "/secure"), ("wrong", "wrong", "/login"), ]) def test_login(page, username, password, expected): page.goto("https://the-internet.herokuapp.com/login") page.locator("#username").fill(username) page.locator("#password").fill(password) page.locator("button[type='submit']").click() assert expected in page.url

‚úÖ One function, multiple test cases! Pytest runs each as a separate test.

Parametrize Syntax

# Single parameter @pytest.mark.parametrize("url", [ "https://example.com", "https://playwright.dev", ]) def test_page_loads(page, url): page.goto(url) assert page.title() # Multiple parameters @pytest.mark.parametrize("width,height", [ (1920, 1080), (1366, 768), (375, 812), ]) def test_responsive(page, width, height): page.set_viewport_size({"width": width, "height": height}) page.goto("https://example.com")

Part 5

Markers & Grouping

Organizing and filtering tests

Built-in Markers

import pytest @pytest.mark.skip(reason="Not implemented yet") def test_future_feature(page): pass @pytest.mark.xfail(reason="Known bug #123") def test_known_bug(page): page.goto("https://example.com/broken") assert page.locator("#fixed").is_visible()
Marker Meaning
@pytest.mark.skip Always skip this test
@pytest.mark.xfail Expected to fail (known bug)
@pytest.mark.skipif Skip if condition is true

Custom Markers

# pytest.ini [pytest] markers = smoke: Quick smoke tests regression: Full regression suite slow: Tests that take a long time
# test_login.py import pytest @pytest.mark.smoke def test_login_page_loads(page): page.goto("https://the-internet.herokuapp.com/login") assert page.locator("h2").is_visible() @pytest.mark.smoke @pytest.mark.regression def test_successful_login(page): # ... login test pass @pytest.mark.regression def test_login_error_messages(page): # ... detailed test pass

Running Tests by Marker

# Run only smoke tests pytest -m smoke # Run only regression tests pytest -m regression # Run smoke OR regression pytest -m "smoke or regression" # Run smoke but NOT slow pytest -m "smoke and not slow"

Part 6

Running Tests

Command line options

Basic Pytest Commands

# Run all tests in current directory pytest # Run specific file pytest test_login.py # Run specific test function pytest test_login.py::test_successful_login # Run tests matching keyword pytest -k "login" # Run tests matching keyword expression pytest -k "login and not error"

Useful Flags

Flag What It Does
-v Verbose output (show test names)
-s Show print() output
--headed Show browser window
--slowmo 500 Slow down actions (ms)
--browser chromium Choose browser
--browser-channel chrome Use installed Chrome
-x Stop on first failure
--lf Run only last failed tests

Playwright-specific Flags

# Run with visible browser pytest --headed # Slow down for debugging pytest --headed --slowmo 1000 # Use Firefox instead of Chromium pytest --browser firefox # Use WebKit (Safari engine) pytest --browser webkit # Run on multiple browsers pytest --browser chromium --browser firefox
