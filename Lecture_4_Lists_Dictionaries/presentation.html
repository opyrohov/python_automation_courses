Why Lists & Dictionaries Matter

Lists

  • Multiple Values - Store many items
  • Ordered - Items have positions
  • Mutable - Can be changed
  • Duplicates OK - Same values allowed

Dictionaries

  • Key-Value Pairs - Structured data
  • Fast Lookup - Find by key
  • Flexible - Store any data type
  • Real-World - Like JSON
üí° In Automation: Lists store multiple locators, test data, and results. Dictionaries structure configuration and user data!

Creating Lists

# Empty list empty_list = [] # List with values numbers = [1, 2, 3, 4, 5] fruits = ["apple", "banana", "cherry"] # Mixed types (not recommended but possible) mixed = [1, "hello", 3.14, True] # List of lists (nested) matrix = [ [1, 2, 3], [4, 5, 6], [7, 8, 9] ]
üí° Best Practice: Use lists to store collections of similar items (all numbers, all strings, etc.)

Accessing List Items

fruits = ["apple", "banana", "cherry", "date", "elderberry"] # Access by index (starts at 0) print(fruits[0]) # apple print(fruits[2]) # cherry # Negative indexing (from the end) print(fruits[-1]) # elderberry (last item) print(fruits[-2]) # date (second from last) # Get length print(len(fruits)) # 5
‚ö†Ô∏è Watch Out: IndexError occurs if you try to access an index that doesn't exist!

List Slicing

numbers = [0, 10, 20, 30, 40, 50, 60, 70] # Syntax: list[start:end:step] # Get first 3 items print(numbers[0:3]) # [0, 10, 20] # Get items from index 2 to end print(numbers[2:]) # [20, 30, 40, 50, 60, 70] # Get last 3 items print(numbers[-3:]) # [50, 60, 70] # Get every other item print(numbers[::2]) # [0, 20, 40, 60] # Reverse a list print(numbers[::-1]) # [70, 60, 50, 40, 30, 20, 10, 0]

List Methods - Adding Items

fruits = ["apple", "banana"] # append() - Add to end fruits.append("cherry") print(fruits) # ["apple", "banana", "cherry"] # insert() - Add at specific position fruits.insert(1, "mango") print(fruits) # ["apple", "mango", "banana", "cherry"] # extend() - Add multiple items fruits.extend(["date", "elderberry"]) print(fruits) # ["apple", "mango", "banana", "cherry", "date", "elderberry"]
üí° Tip: append() adds one item, extend() adds multiple items from another list

List Methods - Removing Items

fruits = ["apple", "banana", "cherry", "banana"] # remove() - Remove first occurrence fruits.remove("banana") print(fruits) # ["apple", "cherry", "banana"] # pop() - Remove and return last item (or by index) last = fruits.pop() print(last) # "banana" print(fruits) # ["apple", "cherry"] # clear() - Remove all items fruits.clear() print(fruits) # []

List Methods - Other Operations

numbers = [3, 1, 4, 1, 5, 9] # sort() - Sort in place numbers.sort() print(numbers) # [1, 1, 3, 4, 5, 9] # reverse() - Reverse in place numbers.reverse() print(numbers) # [9, 5, 4, 3, 1, 1] # count() - Count occurrences print(numbers.count(1)) # 2 # index() - Find position of item print(numbers.index(5)) # 1

List Comprehensions (Preview)

# Traditional way squares = [] for i in range(1, 6): squares.append(i ** 2) print(squares) # [1, 4, 9, 16, 25] # List comprehension way (more Pythonic) squares = [i ** 2 for i in range(1, 6)] print(squares) # [1, 4, 9, 16, 25]

Introduction to Dictionaries

What is a Dictionary?

  • Collection of key-value pairs
  • Keys must be unique
  • Values can be any type
  • Unordered (before Python 3.7)
  • Insertion ordered (Python 3.7+)

When to Use?

  • Store related information
  • Configuration settings
  • User data (like JSON)
  • Test data with labels
  • Fast lookups by key
# Simple dictionary user = { "name": "John", "age": 30, "email": "john@example.com" }

Creating Dictionaries

# Empty dictionary empty_dict = {} # Dictionary with values person = { "name": "Alice", "age": 25, "city": "New York" } # Using dict() constructor car = dict(brand="Toyota", model="Camry", year=2023) # Dictionary with mixed value types test_data = { "username": "testuser", "password": "secret123", "is_active": True, "login_attempts": 0 }

Accessing Dictionary Values

person = { "name": "Alice", "age": 25, "city": "New York" } # Access using bracket notation print(person["name"]) # Alice # get() method (safer - doesn't error if key missing) print(person.get("age")) # 25 print(person.get("phone")) # None print(person.get("phone", "Not found")) # Not found
‚ö†Ô∏è Watch Out: Using brackets with non-existent key raises KeyError. Use .get() to avoid this!

Modifying Dictionaries

person = {"name": "Alice", "age": 25} # Add new key-value pair person["city"] = "New York" # Modify existing value person["age"] = 26 # Update multiple values at once person.update({"email": "alice@example.com", "phone": "555-1234"}) print(person) # {'name': 'Alice', 'age': 26, 'city': 'New York', # 'email': 'alice@example.com', 'phone': '555-1234'}

Dictionary Methods

person = {"name": "Alice", "age": 25, "city": "NYC"} # Get all keys print(person.keys()) # dict_keys(['name', 'age', 'city']) # Get all values print(person.values()) # dict_values(['Alice', 25, 'NYC']) # Get all key-value pairs print(person.items()) # dict_items([('name', 'Alice'), ('age', 25), ('city', 'NYC')]) # Remove a key age = person.pop("age") print(age) # 25 print(person) # {'name': 'Alice', 'city': 'NYC'}

Looping Through Dictionaries

person = {"name": "Alice", "age": 25, "city": "NYC"} # Loop through keys for key in person: print(key) # name # age # city # Loop through values for value in person.values(): print(value) # Loop through both keys and values for key, value in person.items(): print(f"{key}: {value}") # name: Alice # age: 25 # city: NYC

Nested Dictionaries

# Dictionary containing dictionaries users = { "user1": { "name": "Alice", "age": 25, "city": "NYC" }, "user2": { "name": "Bob", "age": 30, "city": "LA" } } # Access nested values print(users["user1"]["name"]) # Alice print(users["user2"]["age"]) # 30

When to Use Lists vs Dictionaries?

Use Lists When:

  • Order matters
  • Items are similar
  • Need to access by position
  • Need duplicates
  • Simple collection needed
# Good use of list urls = [ "page1.com", "page2.com", "page3.com" ]

Use Dictionaries When:

  • Need key-value pairs
  • Fast lookup by key
  • Structured data
  • Each item has properties
  • Like JSON data
# Good use of dict config = { "timeout": 5000, "headless": True, "url": "test.com" }

Combining Lists and Dictionaries

# List of dictionaries - Very common pattern! test_users = [ { "username": "user1", "password": "pass1", "role": "admin" }, { "username": "user2", "password": "pass2", "role": "user" }, { "username": "user3", "password": "pass3", "role": "guest" } ] # Test login with all users for user in test_users: page.fill("#username", user["username"]) page.fill("#password", user["password"]) page.click("#login") # Verify role-specific access

Checking if Key Exists

config = { "timeout": 5000, "headless": True } # Check if key exists if "timeout" in config: print("Timeout is configured") if "url" not in config: print("URL not found, using default") config["url"] = "https://default.com" # Safe access with get() timeout = config.get("timeout", 3000) # Use 3000 if not found
üí° Best Practice: Always check if a key exists before accessing, or use .get() with a default value!

Copying Dictionaries

original = {"name": "Alice", "age": 25} # Wrong way - creates reference, not copy! wrong_copy = original wrong_copy["age"] = 30 print(original["age"]) # 30 (original changed!) # Right way - use copy() original = {"name": "Alice", "age": 25} correct_copy = original.copy() correct_copy["age"] = 30 print(original["age"]) # 25 (original unchanged)
‚ö†Ô∏è Watch Out: Assignment (=) creates a reference, not a copy! Use .copy() to create an independent copy.

Key Takeaways

Lists

  • Ordered collections
  • Access by index [0]
  • Mutable (can change)
  • Methods: append, remove, pop, sort
  • Perfect for sequences

Dictionaries

  • Key-value pairs
  • Access by key ["name"]
  • Fast lookups
  • Methods: get, keys, values, items
  • Perfect for structured data
üí° Remember: Lists for ordered collections, Dictionaries for labeled data!
