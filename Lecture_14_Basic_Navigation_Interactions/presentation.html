Part 1

Navigation Basics

goto, go_back, go_forward, reload

Part 1: Navigation Basics

Navigation is how you control the browser to visit URLs, move through history, and reload pages.

Key Navigation Methods:

  • page.goto(url) - Navigate to a URL
  • page.go_back() - Go back in history
  • page.go_forward() - Go forward in history
  • page.reload() - Reload current page

page.goto() - Navigate to URL

# Basic navigation page.goto("https://example.com") # With options page.goto("https://example.com", wait_until="networkidle", # Wait for network to be idle timeout=30000) # 30 seconds timeout

wait_until Options:

load Wait for 'load' event (default)
domcontentloaded Wait for DOM to be loaded
networkidle Wait until no network activity for 500ms
commit Wait for navigation to commit

goto() Practical Examples

# Example 1: Simple navigation from playwright.sync_api import sync_playwright with sync_playwright() as p: browser = p.chromium.launch(headless=False) page = browser.new_page() # Navigate to website page.goto("https://playwright.dev") print(f"Navigated to: {page.url}") browser.close()
# Example 2: With error handling try: response = page.goto("https://example.com") print(f"Status code: {response.status}") print(f"OK: {response.ok}") # True if status 200-299 except Exception as e: print(f"Navigation failed: {e}")

Browser History Navigation

# Navigate through browser history page.goto("https://example.com") print(f"Page 1: {page.url}") page.goto("https://playwright.dev") print(f"Page 2: {page.url}") # Go back page.go_back() print(f"After back: {page.url}") # https://example.com # Go forward page.go_forward() print(f"After forward: {page.url}") # https://playwright.dev
Note: go_back() and go_forward() will raise an error if there's no history to navigate to. Always ensure there's a valid history entry.

page.reload() - Refresh Page

# Basic reload page.goto("https://example.com") page.reload() # Reload with options page.reload(wait_until="networkidle", timeout=30000)

Common Use Cases:

  • ‚úì Testing dynamic content updates
  • ‚úì Verifying page state after changes
  • ‚úì Resetting form data
  • ‚úì Testing page load performance
# Example: Verify content after reload page.goto("https://example.com") title_before = page.title() page.reload() title_after = page.title() assert title_before == title_after

Complete Navigation Example

from playwright.sync_api import sync_playwright import time with sync_playwright() as p: browser = p.chromium.launch(headless=False, slow_mo=1000) page = browser.new_page() # 1. Navigate to first page page.goto("https://example.com") print(f"1. Current: {page.url}") time.sleep(1) # 2. Navigate to second page page.goto("https://playwright.dev") print(f"2. After forward: {page.url}") time.sleep(1) # 5. Reload page.reload() print(f"5. After reload: {page.url}") browser.close()

Part 2

Pages & Browser Contexts

Understanding Browser ‚Üí Context ‚Üí Page hierarchy

Part 2: Pages & Browser Contexts

Browser Context: An isolated incognito-like session within a browser instance. Think of it as a separate browser profile.

üåê Browser

  • The browser process (Chrome, Firefox, etc.)
  • Heavy to create
  • Contains contexts

üìÇ Context

  • Isolated session (cookies, storage)
  • Lightweight
  • Contains pages

üìÑ Page

  • A single tab/window
  • Where you interact with content
  • Belongs to a context

Browser ‚Üí Context ‚Üí Page

from playwright.sync_api import sync_playwright with sync_playwright() as p: # 1. Launch Browser browser = p.chromium.launch(headless=False) # 2. Create Context (automatic with new_page) page = browser.new_page() # Creates default context + page # 3. Use Page page.goto("https://example.com") browser.close()
Hierarchy: 
Browser (Chrome)

    ‚îú‚îÄ‚îÄ Page 1 (Tab 1)
    ‚îú‚îÄ‚îÄ Page 2 (Tab 2)
    ‚îî‚îÄ‚îÄ Page 3 (Tab 3)
‚îî‚îÄ‚îÄ Context 2 (Different session)

Multiple Browser Contexts

from playwright.sync_api import sync_playwright with sync_playwright() as p: browser = p.chromium.launch(headless=False) # Context 1: User A context1 = browser.new_context() page1 = context1.new_page() page1.goto("https://example.com") # Context 2: User B (separate session) context2 = browser.new_context() page2 = context2.new_page() page2.goto("https://example.com") # Each context has isolated cookies/storage! context1.close() context2.close() browser.close()
Use Case: Testing multi-user scenarios, different login states, or isolated test environments.

Multiple Pages in One Context

from playwright.sync_api import sync_playwright with sync_playwright() as p: browser = p.chromium.launch(headless=False) context = browser.new_context() # Create multiple pages (tabs) page1 = context.new_page() page1.goto("https://example.com") print(f"Page 1: {page1.title()}") page2 = context.new_page() page2.goto("https://playwright.dev") print(f"Page 2: {page2.title()}") page3 = context.new_page() page3.goto("https://github.com") print(f"Page 3: {page3.title()}") # All pages share same context (cookies, storage) print(f"Total pages: {len(context.pages)}") # 3 browser.close()

Managing Multiple Pages

# Get all pages in context all_pages = context.pages print(f"Total pages: {len(all_pages)}") # Iterate through pages for i, page in enumerate(all_pages): print(f"Page {i+1}: {page.url}") # Switch between pages page1.bring_to_front() # Bring to front (focus) # Close specific page page2.close() # Get current page current_page = context.pages[0]
Important: When you close a context, ALL pages in that context are automatically closed.

Browser Context Options

# Create context with options context = browser.new_context( viewport={'width': 1920, 'height': 1080}, user_agent='Custom User Agent', locale='en-US', timezone_id='America/New_York', permissions=['geolocation'], geolocation={'latitude': 40.7128, 'longitude': -74.0060}, ignore_https_errors=True, java_script_enabled=True )

Common Options:

  • viewport - Browser window size
  • user_agent - Custom user agent string
  • locale - Language/region (en-US, fr-FR, etc.)
  • timezone_id - Timezone
  • permissions - Grant permissions (notifications, geolocation)
  • ignore_https_errors - Ignore SSL errors

Part 3

Basic Actions

click(), fill(), type()

Part 3: Basic Actions

Actions are how you interact with elements on the page - clicking buttons, filling forms, typing text.

Three Core Actions:

Action Purpose When to Use
click() Click an element Buttons, links, checkboxes
fill() Fill input fast Forms (fast, no events)
type() Type character by character When you need keyboard events

page.click() - Click Elements

# Basic click page.click("button") # Click with selector page.click("#submit-btn") page.click(".login-button") page.click("text=Sign In") # Click with options page.click("button", button="right", # Right-click click_count=2, # Double-click timeout=5000) # 5 second timeout

Click Options:

  • button - "left" (default), "right", "middle"
  • click_count - Number of clicks (1=single, 2=double)
  • delay - Delay between mousedown and mouseup (ms)
  • force - Skip actionability checks (use carefully!)
  • timeout - Maximum wait time

Click Examples

from playwright.sync_api import sync_playwright with sync_playwright() as p: browser = p.chromium.launch(headless=False) page = browser.new_page() page.goto("https://example.com") # Click by different selectors page.click("button#submit") # ID page.click("button.primary") # Class page.click("text=Click Me") # Text content page.click("button:has-text('Save')") # Contains text # Double-click page.click("button", click_count=2) # Right-click page.click("button", button="right") browser.close()

page.fill() - Fill Input Fields

# Basic fill page.fill("input[name='username']", "john_doe") page.fill("#email", "john@example.com") page.fill(".password-input", "SecurePass123") # Fill clears existing value first page.fill("input", "New Value") # Old value is cleared
How fill() works:
  1. Focuses the input element
  2. Clears existing value
  3. Sets the new value instantly
  4. Does NOT trigger keyboard events
Use fill() when: You want to quickly fill forms without triggering keyboard events. It's faster than type().

page.type() - Type Text Character by Character

# Basic type page.type("input[name='search']", "Playwright") # Type with delay page.type("input", "Hello World", delay=100) # 100ms between chars # Type triggers keyboard events page.type("input", "Test") # Fires keydown, keypress, keyup
How type() works:
  1. Focuses the input element
  2. Types each character one by one
  3. Triggers keyboard events (keydown, keypress, keyup)
  4. Slower but more realistic
Use type() when: Your application listens to keyboard events or you need to simulate human typing.

fill() vs type() - Key Differences

Feature fill() type()
Speed ‚ö° Very Fast üê¢ Slower
Clears First ‚úÖ Yes ‚ùå No (appends)
Keyboard Events ‚ùå No ‚úÖ Yes
Use Case Forms, inputs Search boxes, autocomplete
Recommended ‚úÖ Default choice ‚ö†Ô∏è Special cases
# Choose fill() for most cases page.fill("input[name='email']", "user@example.com") # Fast ‚úì # Use type() for autocomplete/search page.type("input[name='search']", "Python", delay=50) # Triggers events ‚úì

Part 4: Getting Page Information

Page Information: Extract useful data from the current page like title, URL, and content.

from playwright.sync_api import sync_playwright with sync_playwright() as p: browser = p.chromium.launch() page = browser.new_page() page.goto("https://example.com") # Get page title title = page.title() print(f"Title: {title}") # Get current URL url = page.url print(f"URL: {url}") # Get page HTML content html = page.content() print(f"HTML length: {len(html)}") browser.close()

page.title() - Get Page Title

# Get page title title = page.title() print(title) # "Example Domain" # Use in assertions assert page.title() == "Example Domain" assert "Example" in page.title() # Check title across pages page.goto("https://example.com") title1 = page.title() page.goto("https://playwright.dev") title2 = page.title() print(f"Title 1: {title1}") print(f"Title 2: {title2}")
Common Use Case: Verify navigation was successful by checking if the page title matches expectations.

page.url - Get Current URL

# Get current URL current_url = page.url print(current_url) # "https://example.com/" # Check URL after navigation page.goto("https://example.com/login") print(page.url) # "https://example.com/login" # Use in assertions assert "login" in page.url assert page.url.startswith("https://") # Track URL changes page.goto("https://example.com") url_before = page.url page.click("a[href='/about']") url_after = page.url print(f"Before: {url_before}") print(f"After: {url_after}")

page.content() - Get Page HTML

# Get full page HTML html = page.content() print(html[:100]) # Print first 100 chars # Check if text exists in HTML if "Welcome" in html: print("Welcome message found!") # Save HTML to file with open("page.html", "w", encoding="utf-8") as f: f.write(page.content()) # Get HTML length print(f"Page size: {len(html)} characters")
Note: page.content() returns the full HTML including dynamically loaded content. It's the entire DOM state at that moment.

Closing Browsers & Pages

Proper Cleanup: Always close pages, contexts, and browsers to free resources and prevent memory leaks.

Closing Hierarchy:

# Close page only page.close() # Close context (closes all its pages) context.close() # Close browser (closes all contexts and pages) browser.close()
Remember:
  • browser.close() ‚Üí Closes everything
  • context.close() ‚Üí Closes all pages in that context
  • page.close() ‚Üí Closes only that specific page

Closing Examples

# Example 1: Close individual page page1 = browser.new_page() page2 = browser.new_page() page1.close() # Close only page1 page2.close() # Close only page2 browser.close() # Close browser
# Example 2: Close context (closes all pages) context = browser.new_context() page1 = context.new_page() page2 = context.new_page() context.close() # Closes both page1 and page2 browser.close()
# Example 3: Close browser (closes everything) browser = p.chromium.launch() page1 = browser.new_page() page2 = browser.new_page() browser.close() # Closes all contexts and pages

Context Managers - Automatic Cleanup

# Recommended: Use 'with' statement from playwright.sync_api import sync_playwright with sync_playwright() as p: browser = p.chromium.launch() page = browser.new_page() page.goto("https://example.com") print(page.title()) # browser.close() called automatically!
Why use 'with'?
  • ‚úÖ Automatic cleanup even if errors occur
  • ‚úÖ Cleaner code
  • ‚úÖ Prevents resource leaks
  • ‚úÖ Best practice for production code

Error Handling with Proper Cleanup

from playwright.sync_api import sync_playwright # Bad: Browser might not close on error p = sync_playwright().start() browser = p.chromium.launch() page = browser.new_page() page.goto("https://invalid-url") # Error! Browser not closed browser.close()
# Good: Browser always closes with sync_playwright() as p: browser = p.chromium.launch() page = browser.new_page() try: page.goto("https://invalid-url") except Exception as e: print(f"Error: {e}") finally: browser.close() # Always closes
# Best: Context manager handles everything with sync_playwright() as p: browser = p.chromium.launch() page = browser.new_page() try: page.goto("https://invalid-url") except Exception as e: print(f"Error: {e}") # Automatic cleanup!

Best Practices

‚úÖ DO:

  • Use with sync_playwright() for automatic cleanup
  • Use fill() for forms (faster)
  • Use type() only when keyboard events are needed
  • Check page.url or page.title() after navigation
  • Close pages/contexts when done
  • Use meaningful timeouts
  • Handle navigation errors with try/except

‚ùå DON'T:

  • Forget to close browsers (memory leaks!)
  • Use type() everywhere (slower)
  • Ignore navigation failures
  • Use force=True unless necessary
  • Create too many contexts (resource intensive)

Common Errors & Solutions

Error Cause Solution
TimeoutError Element not found/visible Check selector, increase timeout
Navigation failed Invalid URL or network issue Verify URL, check network
Element not clickable Element covered or disabled Wait for visibility, check CSS
fill() doesn't work Not an input element Use correct selector, verify element type
Browser not closing Exception before close() Use context manager or try/finally

Waiting Strategies

Playwright Auto-waits: Most actions automatically wait for elements to be actionable (visible, enabled, stable).

# Automatic waiting (recommended) page.click("button") # Waits for button to be clickable page.fill("input", "text") # Waits for input to be fillable # Explicit waiting (when needed) page.wait_for_selector("div.content") page.wait_for_url("**/dashboard") page.wait_for_load_state("networkidle") # Wait for specific condition page.wait_for_function("() => document.readyState === 'complete'") # Timeout options page.click("button", timeout=5000) # 5 seconds page.wait_for_selector(".element", timeout=10000) # 10 seconds
Best Practice: Trust Playwright's auto-waiting. Only add explicit waits when necessary.
