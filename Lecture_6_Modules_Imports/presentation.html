What are Modules?

A module is a Python file containing functions, classes, and variables that you can import and use in other programs.

Why Use Modules?

  • ‚úÖ Code organization
  • ‚úÖ Reusability
  • ‚úÖ Avoid repetition
  • ‚úÖ Easier testing
  • ‚úÖ Team collaboration

Types

  • Module: Single .py file
  • Package: Folder with modules
  • Library: Collection of packages

Importing Modules

# Method 1: Import entire module import math print(math.sqrt(16)) # 4.0 print(math.pi) # 3.14159... print(math.ceil(4.2)) # 5
Syntax: import module_name
Usage: module_name.function()

Import Specific Items

# Method 2: Import specific items from math import sqrt, pi print(sqrt(25)) # 5.0 - no need for math. prefix print(pi) # 3.14159... # Import multiple items from math import sqrt, pi, ceil, floor print(sqrt(16)) print(ceil(4.3)) print(floor(4.8))
üí° Benefit: Cleaner code - no module prefix needed!

Import with Alias

# Method 3: Import with alias import datetime as dt now = dt.datetime.now() print(now) today = dt.date.today() print(today) # Import specific item with alias from datetime import datetime as dt_now print(dt_now.now())
When to use aliases:
  • Long module names
  • Avoid naming conflicts
  • Convention (e.g., numpy as np, pandas as pd)

Import Patterns Comparison

# Pattern 1: Import module import random print(random.randint(1, 10)) # Pattern 2: Import specific function from random import randint print(randint(1, 10)) # Pattern 3: Import with alias import random as rnd print(rnd.randint(1, 10)) # Pattern 4: Import all (NOT recommended!) from random import * # Avoid this!
‚ö†Ô∏è Avoid: from module import *
It imports everything and can cause naming conflicts

Creating Your Own Module

# greetings.py - your custom module def say_hello(name): return f"Hello, {name}!" def say_goodbye(name): return f"Goodbye, {name}!" MODULE_VERSION = "1.0.0" # main.py - using your module from greetings import say_hello, say_goodbye print(say_hello("Alice")) print(say_goodbye("Bob"))
That's it! Any .py file is a module you can import

The __name__ Variable

# calculator.py def add(a, b): return a + b def multiply(a, b): return a * b # This only runs when file is executed directly if __name__ == "__main__": print("Testing calculator module") print(add(5, 3)) print(multiply(4, 6))
__name__ is:
  • "__main__" when file is run directly
  • Module name when imported

datetime Module

from datetime import datetime, timedelta # Current date and time now = datetime.now() print(now) print(now.strftime("%Y-%m-%d %H:%M:%S")) # Date arithmetic today = datetime.now().date() tomorrow = today + timedelta(days=1) next_week = today + timedelta(weeks=1) print(f"Today: {today}") print(f"Tomorrow: {tomorrow}") print(f"Next week: {next_week}")
Perfect for: Timestamps, test data, time calculations

random Module

from random import randint, choice, shuffle # Random integer print(randint(1, 100)) # Random between 1-100 # Random choice browsers = ["Chrome", "Firefox", "Safari"] print(choice(browsers)) # Shuffle list numbers = [1, 2, 3, 4, 5] shuffle(numbers) print(numbers)
Great for: Generating test data, randomizing test order

json Module

import json # Python dict to JSON string user = { "name": "Alice", "email": "alice@example.com", "age": 25 } json_string = json.dumps(user, indent=2) print(json_string) # JSON string to Python dict json_data = '{"name": "Bob", "score": 95}' data = json.loads(json_data) print(data["name"]) # Bob
Use for: API testing, configuration files, test data

pathlib Module (Modern Path Handling)

from pathlib import Path # Current file path current_file = Path(__file__) print(current_file.name) # File name print(current_file.parent) # Parent directory # Build paths (works on any OS!) project_root = Path("/project") tests_dir = project_root / "tests" test_file = tests_dir / "test_login.py" # Check properties print(current_file.exists()) print(current_file.is_file())
Benefit: Cross-platform path handling!

Installing Packages with pip

pip is Python's package installer
It downloads packages from PyPI (Python Package Index)
# Install a package pip install requests # Install specific version pip install playwright==1.40.0 # Install multiple packages pip install requests pytest playwright # Upgrade a package pip install --upgrade requests # Uninstall a package pip uninstall requests

Essential pip Commands

# Check pip version pip --version # List installed packages pip list # Show package details pip show playwright # Check outdated packages pip list --outdated # Freeze current packages pip freeze > requirements.txt
üí° Tip: If pip doesn't work, try python -m pip

requirements.txt File

# requirements.txt - list all project dependencies # Testing framework pytest==7.4.3 pytest-playwright==0.4.3 # Playwright for browser automation playwright==1.40.0 # HTTP requests requests==2.31.0 # Test data generation faker==20.1.0
# Install all packages from requirements.txt pip install -r requirements.txt
Best Practice: Always use requirements.txt for projects!

Installing Playwright

Two steps required:
# Step 1: Install the Python package pip install playwright # Step 2: Install browser binaries playwright install # Or install specific browsers playwright install chromium playwright install firefox playwright install webkit # Check installation playwright --version
‚úÖ You're ready for browser automation!

Playwright Import Patterns

# Sync API (we'll use this!) from playwright.sync_api import sync_playwright # Import with expect for assertions from playwright.sync_api import sync_playwright, expect # Import specific classes from playwright.sync_api import Page, Browser, BrowserContext # All-in-one for tests from playwright.sync_api import ( sync_playwright, expect, Page, Browser )

Basic Playwright Script

from playwright.sync_api import sync_playwright def run(): with sync_playwright() as p: # Launch browser browser = p.chromium.launch(headless=False) # Create a page page = browser.new_page() # Navigate page.goto("https://example.com") # Close browser browser.close() if __name__ == "__main__": run()

Page Object Pattern

# pages/login_page.py from playwright.sync_api import Page class LoginPage: def __init__(self, page: Page): self.page = page def navigate(self): self.page.goto("https://example.com/login") def login(self, username, password): self.page.fill("#username", username) self.page.fill("#password", password) self.page.click("#login-btn") # tests/test_login.py from pages.login_page import LoginPage def test_login(page): login_page = LoginPage(page) login_page.navigate() login_page.login("user", "pass")

Creating Helper Functions

# utils/helpers.py from playwright.sync_api import Page from datetime import datetime def take_screenshot(page: Page, name: str): """Take a screenshot with timestamp""" timestamp = datetime.now().strftime("%Y%m%d_%H%M%S") filename = f"screenshots/{name}_{timestamp}.png" page.screenshot(path=filename) return filename def wait_and_click(page: Page, selector: str): """Wait for element and click it""" page.wait_for_selector(selector) page.click(selector) # tests/test_example.py from utils.helpers import take_screenshot, wait_and_click

Configuration Module

# config/settings.py from pathlib import Path # Paths PROJECT_ROOT = Path(__file__).parent.parent SCREENSHOTS_DIR = PROJECT_ROOT / "screenshots" # Test environment BASE_URL = "https://example.com" DEFAULT_TIMEOUT = 30000 # Browser config BROWSER_TYPE = "chromium" HEADLESS = False # Test data TEST_USERS = { "valid": { "username": "testuser@example.com", "password": "Test123!" } }

Module Best Practices

‚úÖ DO

  • One module = one purpose
  • Descriptive names
  • Add docstrings
  • Group related functions
  • Use __init__.py for packages

‚ùå DON'T

  • Huge modules with everything
  • Generic names (utils, helpers)
  • Circular dependencies
  • Use from x import *
  • Forget requirements.txt

Import Best Practices

# Organize imports in this order: # 1. Standard library import json import time from datetime import datetime from pathlib import Path # 2. Third-party libraries import pytest from playwright.sync_api import Page, expect # 3. Your own modules from pages.login_page import LoginPage from utils.helpers import take_screenshot from config.settings import BASE_URL
Clean imports = readable code!
